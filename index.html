Aprendendo GIT Flow

Master

Essa branch é a versão de produção, não programar nela nunca, e só mergear ela com a branch de release ou hotfix.

Hotfix

Essa branch serve para corrigir problemas direto da master, fazer checkout na master, criar a branch hotfix/[versão atual da master], ex: hotfix/v1.0.0, corrigir o bug, fazer merge dela direto na master e na develop, de tiver alguma release para ser publicada, já faz agora tbm, se o bug afeta o alguma feature, pode fzr isso aqui tbm E na master você já cria a nova TAG de versão, depois disso, deletar a branch.

Develop

Essa branch de desenvolvimento geral, todas as novas features desenvolvidas devem ser jogadas aqui e somente aqui, não se faz Merge da develop direto na master, deve ser criada uma branch de release, e deve-se evitar ao máximo commitar aqui também.

Feature

aqui são desenvolvidas as novas features,Uma branch para cada feature, de preferência não fazer muitas modificações aqui, apenas criar. Dps de prontas deve-se fazer o Merge dela na develop e então deletar, n deve fzr merge dela na develop e voltar pra ela, mergear somente quando a feature estiver pronta, e se ela quebra algo, nunca subir, se ela tem alguma dependência já em develop, faz o merge da develop da develop nela, mas agora n precisa mais fzr Merge da develop com a sua antes de fzr Merge na develop, we tiver conflitos na HR de fzr o Merge na develop, CORRIGIR IMEDIATAMENTE.

Release

Essa branch de release deve ser criada quando uma feature está pronta para ser publicada em produção, o padrão é release/[nova versão], ex: release/v1.1.0, aqui vai ser testadas as novas features, não criar novas branchs a partir daqui, se tiver um bug corrigir aqui msm, e fazer Merge dela na develop caso aja novas mudanças, finalizada e testada a release, mandar pra master, criar a nova TAG, e tbm mergear a release na develop, depois disso, deletar a branch.
